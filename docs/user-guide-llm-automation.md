## 1. Введение

### Что умеет система

Система позволяет автоматически создавать учебные материалы на основе ваших лекций. Вы загружаете файл с текстом лекции (или вставляете текст вручную), выбираете, какой тип контента нужен, — и искусственный интеллект (Claude AI) генерирует черновики учебных материалов. После этого вы просматриваете результат, редактируете при необходимости и утверждаете — только после вашего одобрения материал попадает в учебную базу данных.

Это не замена преподавателю, а инструмент, который экономит время на рутинной работе: составлении тестов, карточек препаратов и ситуационных задач.

### Какие типы контента можно генерировать

| Тип | Описание |
|-----|----------|
| **Тестовые вопросы** | Вопросы с множественным выбором (4–6 вариантов ответа). Включают пояснение правильного ответа, уровень сложности и привязку к компетенции. |
| **Карточки препаратов** | Структурированные карточки с названием, группой, формой выпуска, дозировкой, показаниями, противопоказаниями, побочными эффектами и особенностями полевого применения. |
| **Flash-карточки** | Карточки для заучивания — описывают препараты, медицинские приборы, инструменты или оборудование. Удобны для быстрого повторения ключевых фактов. |
| **Клинические сценарии** | Ситуационные задачи: описание боевой обстановки, состояние раненого, варианты действий. Основаны на протоколах TCCC / тактической медицины. |

### Общий принцип работы

**Загрузил файл → выбрал тип → получил черновик → отредактировал → утвердил → материал попал в базу**

## 2. Подготовка материала

### Поддерживаемые форматы файлов

| Формат | Расширение | Примечания |
|--------|-----------|------------|
| **PDF** | `.pdf` | Текстовый PDF. Сканированные документы (изображения) **не поддерживаются** — система не распознаёт текст с картинок. |
| **Word** | `.docx`, `.doc` | Обычный текстовый документ. Сложные таблицы и формулы могут быть извлечены некорректно. |
| **Текстовый файл** | `.txt` | Простой текст в кодировке UTF-8. |

### Ограничения по размеру

- **Максимальный размер файла:** 10 МБ
- **Минимальная длина текста:** 100 символов (иначе система откажет в генерации)
- **Рекомендуемый объём:** от 1 до 30 страниц текста (примерно 3 000–90 000 символов)
- **Максимальный объём текста для обработки:** ~100 000 символов. Если текст длиннее, будут использованы только первые 100 000 символов.

> **Предупреждение:** Если длина текста превышает 80 000 символов, система покажет предупреждение. Для лучшего результата загружайте отдельные главы или разделы, а не весь учебник целиком.

### Что лучше загружать

- **Конспект одной лекции** — идеальный вариант, тема сфокусирована
- **Глава учебника** по конкретной теме
- **Методические указания** к занятию
- **Раздел о конкретном препарате или группе препаратов**

### Рекомендации по качеству файлов

1. **PDF** должен быть текстовым (текст можно выделить и скопировать). Если PDF создан из скана — текст не будет извлечён.
2. **Word-документы** должны содержать обычный текст. Если информация представлена сложными таблицами, лучше переформатировать её в текстовое описание.
3. Чем **структурированнее** текст (заголовки, списки, абзацы), тем качественнее будет результат.
4. Загружайте **один материал по одной теме** за раз, а не весь курс целиком.

## 3. Пошаговая инструкция

### Шаг 1. Откройте раздел «Генерация контента»

1. Войдите в **Кабинет инструктора** (введите свой email и пароль).
2. В боковой панели навигации нажмите **«Генерация контента»**.
3. Откроется страница с заголовком **«Генерация контента»** и подсказкой: *«Загрузите материалы лекции — Claude AI сгенерирует учебный контент»*.

[СКРИНШОТ: Боковая панель навигации с выделенным пунктом «Генерация контента»]

В верхней части страницы вы увидите подсказку из трёх шагов:
1. Загрузите PDF / DOCX или вставьте текст
2. Выберите типы контента и настройки
3. Ревьюите, редактируйте и утверждайте

---

### Шаг 2. Загрузите файл с текстом лекции

В блоке **«Загрузка материалов»** есть два способа предоставить текст:

#### Способ А: Загрузка файла (рекомендуется)

1. Нажмите на область загрузки с надписью **«Перетащите файл сюда или нажмите для выбора»** — откроется окно выбора файла.
2. Или просто **перетащите** файл из проводника прямо на эту область.
3. Система примет файлы форматов PDF, DOCX или TXT (до 10 МБ).

[СКРИНШОТ: Зона загрузки файлов с иконкой документа]

После загрузки система:
- Извлечёт текст из файла (вы увидите надпись *«Извлечение текста...»*)
- Покажет информацию о файле: имя, размер в МБ, количество страниц
- Заполнит поле ввода извлечённым текстом (оно станет неактивным для редактирования)
- Покажет статистику: количество извлечённых символов и примерное число страниц

Если файл не подошёл, нажмите кнопку **✕** (крестик) рядом с именем файла, чтобы очистить и загрузить другой.

#### Способ Б: Вставка текста вручную

Если у вас нет файла нужного формата, можно скопировать текст из любого источника и вставить его в поле **«Вставьте текст лекции»** под разделителем *«или»*.

---

### Шаг 3. Настройте параметры генерации

В блоке **«Настройки генерации»** задайте параметры:

#### Тип контента

Выберите один или несколько типов (все отмечены по умолчанию):

- **Тестовые вопросы** — вопросы с вариантами ответа для проверки знаний
- **Карточки препаратов** — структурированная информация о препаратах из текста лекции
- **Flash-карточки** — краткие карточки для заучивания (препараты, инструменты, оборудование)
- **Сценарии** — ситуационные задачи для отработки принятия решений в боевых условиях

> Должен быть выбран **хотя бы один** тип контента.

[СКРИНШОТ: Чекбоксы выбора типов контента]

#### Компетенция

В выпадающем списке **«Компетенция»** выберите компетенцию из учебного плана, к которой будут привязаны сгенерированные вопросы и сценарии. Компетенции сгруппированы по блокам (например, «Шок и неотложные состояния», «Фармакология» и др.).

Если компетенция не важна, оставьте значение **«-- Не указана --»**.

Поле **«Блок»** заполняется автоматически на основе выбранной компетенции.

#### Сложность

Выберите уровень сложности генерируемых вопросов и сценариев:

| Уровень | Что проверяет |
|---------|---------------|
| **1 — Базовый** | Знание фактов (определения, названия, дозировки) |
| **2 — Средний** (по умолчанию) | Понимание материала (связи, причины, механизмы) |
| **3 — Продвинутый** | Анализ и применение (принятие решений, нетипичные ситуации) |

#### Количество

Укажите, сколько элементов **каждого** выбранного типа нужно сгенерировать. Допустимое значение: от **1 до 20**. По умолчанию — **5**.

> **Пример:** Если выбрано 3 типа контента и количество = 5, система сгенерирует до 15 элементов (по 5 каждого типа).

#### Дополнительные пожелания (необязательно)

В текстовом поле **«Дополнительные пожелания»** вы можете дать ИИ подсказку, на чём сделать акцент. Примеры:

- *«Фокус на дозировках»*
- *«Сложные вопросы про полевые условия»*
- *«Включить вопросы по побочным эффектам НПВС»*
- *«Сценарии с множественными ранениями»*

---

### Шаг 4. Запустите генерацию

1. Убедитесь, что текст загружен (минимум 100 символов) и выбран хотя бы один тип контента.
2. Нажмите кнопку **«Сгенерировать контент»**.

> Кнопка неактивна (серая), пока не загружен текст достаточной длины и не выбран хотя бы один тип контента.

[СКРИНШОТ: Кнопка «Сгенерировать контент» активна, настройки заполнены]

#### Что происходит во время генерации

- Появляется индикатор с вращающимся спиннером и надписью: **«Claude AI генерирует контент...»**
- Под ним подсказка: *«Это может занять 15–60 секунд в зависимости от объёма текста»*
- Предупреждение: **«Не закрывайте страницу»**

**Ориентировочное время ожидания:**

| Объём | Типы | Примерное время |
|-------|------|-----------------|
| 1–5 страниц, 1 тип | 15–25 секунд |
| 5–15 страниц, 2–3 типа | 25–45 секунд |
| 15–30 страниц, 4 типа | 40–60 секунд |

> **Важно:** Не закрывайте и не перезагружайте страницу, пока идёт генерация. Если прошло более 90 секунд, система покажет сообщение об ошибке (таймаут).

---

### Шаг 5. Просмотрите и отредактируйте результаты

После завершения генерации появится блок **«Результаты генерации»**.

#### Статистика

В верхней части результатов показана статистика:
- **Время генерации** (в секундах)
- **Количество токенов** (единиц обработки текста — вход и выход)
- **Примерная стоимость** запроса
- **Количество** сгенерированных элементов

[СКРИНШОТ: Блок статистики генерации — время, токены, стоимость, количество]

#### Карточки результатов

Сгенерированные элементы отображаются в виде **карточек**, сгруппированных по типу:
- Тестовые вопросы
- Сценарии
- Карточки препаратов
- Flash-карточки

Каждая карточка имеет:
- **Заголовок** с типом (например, «Тестовый вопрос», «Сценарий», «Препарат», «Flash-карточка»)
- **Статус**: `draft` (черновик), `approved` (утверждён) или `rejected` (отклонён)
- **Содержимое** — зависит от типа:
  - Для вопросов: текст вопроса, варианты ответов (правильные отмечены галочкой), пояснение
  - Для препаратов: название, группа, форма, дозировка, показания, противопоказания, побочные эффекты, полевые особенности
  - Для flash-карточек: тип (препарат/прибор/инструмент/оборудование) + все поля карточки
  - Для сценариев: описание ситуации, вопрос, варианты действий, обоснование

[СКРИНШОТ: Пример карточки тестового вопроса со статусом «draft» и кнопками действий]

#### Действия с отдельной карточкой

Под каждой карточкой со статусом «черновик» находятся три кнопки:

| Кнопка | Действие |
|--------|----------|
| **«Редактировать»** | Открывает режим редактирования прямо внутри карточки |
| **«Утвердить»** | Одобряет элемент и переносит его в учебную базу данных |
| **«Отклонить»** | Помечает элемент как отклонённый (он не попадёт в базу) |

#### Редактирование

При нажатии **«Редактировать»** содержимое карточки превращается в форму с полями для ввода:
- Для вопросов: можно изменить текст вопроса, варианты ответов, отметить правильные, изменить пояснение, сложность и категорию.
- Для препаратов и flash-карточек: можно изменить все поля (название, группу, форму, дозировку и т. д.).
- Для сценариев: можно изменить описание ситуации, вопрос, варианты действий и обоснование.

После редактирования нажмите **«Сохранить»** для сохранения изменений или **«Отмена»** для возврата к исходному виду. После сохранения карточка остаётся в статусе «черновик» — вы можете далее утвердить или отклонить её.

[СКРИНШОТ: Карточка вопроса в режиме редактирования — поля ввода, кнопки «Сохранить» и «Отмена»]

#### Массовые действия

Над списком карточек находятся кнопки для массовых операций:
- **«Утвердить все»** — утверждает все карточки со статусом «черновик» за один раз
- **«Отклонить все»** — отклоняет все черновики

Рядом отображается счётчик: *«Всего: N | Черновики: N | Утверждено: N | Отклонено: N»*

> **Рекомендация:** Перед массовым утверждением обязательно просмотрите все карточки. Лучше утверждать по одной, проверяя медицинскую точность каждого элемента.

---

### Шаг 6. Куда попадают утверждённые элементы

После нажатия **«Утвердить»** элемент автоматически переносится в учебную базу данных:

| Тип контента | Куда попадает | Где найти в кабинете |
|--------------|---------------|----------------------|
| **Тестовый вопрос** | Таблица «Вопросы» | Раздел **«Вопросы»** в боковом меню |
| **Клинический сценарий** | Таблица «Вопросы» (с категорией «scenario») | Раздел **«Вопросы»** в боковом меню |
| **Карточка препарата** | Таблица «Препараты» | Раздел **«Препараты»** в боковом меню |
| **Flash-карточка** | Таблица «Препараты» (с пометкой типа элемента) | Раздел **«Препараты»** в боковом меню |

После утверждения:
1. Кнопки действий на карточке исчезают, статус меняется на `approved`.
2. Элемент появляется в соответствующем разделе кабинета инструктора (Вопросы или Препараты), где его можно дополнительно отредактировать.
3. При наличии синхронизации с Google Sheets элемент будет также автоматически перенесён в таблицу.

> Отклонённые элементы (`rejected`) **не попадают** в учебную базу. Они сохраняются в истории генерации для справки.

## 4. Советы и лучшие практики

1. **Генерируйте по одной теме за раз.** Не загружайте весь курс целиком — качество результата будет выше, если текст сфокусирован на одной теме.

2. **Всегда проверяйте медицинскую точность.** ИИ может допустить ошибку в дозировке, противопоказании или описании протокола. Вы как специалист — последняя инстанция проверки.

3. **Редактируйте формулировки под свой стиль.** Сгенерированный текст может быть «книжным». Адаптируйте его под ваш стиль преподавания и уровень подготовки курсантов.

4. **Используйте поле «Пояснение».** Для тестовых вопросов и сценариев обязательно проверьте и при необходимости доработайте пояснение (explanation / обоснование). Оно помогает курсантам при разборе ошибок.

5. **Задавайте дополнительные пожелания.** Поле «Дополнительные пожелания» помогает направить ИИ: укажите конкретные темы, препараты, типы ситуаций или формат, который вам нужен.

6. **Если результат неудовлетворительный — попробуйте ещё раз.** ИИ генерирует разные варианты каждый раз. Также попробуйте:
   - Загрузить более структурированный текст
   - Уменьшить количество типов контента за один раз
   - Добавить более конкретные пожелания
   - Изменить уровень сложности

7. **Начните с малого количества.** При первом использовании выберите 2–3 элемента одного типа, чтобы оценить качество, а затем увеличивайте количество.

8. **Не утверждайте массово без проверки.** Кнопка «Утвердить все» удобна, когда вы уже проверили все карточки. Не используйте её «вслепую».

## 5. Типичные проблемы и решения

### Файл не загружается

| Проблема | Причина | Решение |
|----------|---------|---------|
| *«Неподдерживаемый формат»* | Файл не в формате PDF, DOCX или TXT | Сохраните документ в одном из поддерживаемых форматов |
| *«Файл слишком большой (X МБ). Максимум: 10 МБ»* | Размер файла превышает 10 МБ | Разделите файл на части или уберите изображения из документа |
| Текст не извлекается из PDF | PDF содержит сканированные страницы (изображения) | Используйте текстовый PDF или скопируйте текст вручную и вставьте в поле ввода |

### Ошибки при генерации

| Сообщение | Что произошло | Что делать |
|-----------|---------------|------------|
| *«Генерация заняла слишком много времени»* | Сервер не успел обработать запрос за 90 секунд | Загрузите меньше текста или выберите меньше типов контента. Нажмите **«Попробовать снова»** |
| *«Превышен лимит запросов»* | Слишком много запросов к ИИ за короткое время | Подождите 1–2 минуты и попробуйте снова |
| *«Сервер Claude перегружен»* | Временная перегрузка сервера ИИ | Попробуйте через несколько минут |
| *«Claude вернул некорректный ответ»* | ИИ сгенерировал данные в неожиданном формате | Нажмите **«Попробовать снова»** — каждая генерация уникальна, следующая попытка может быть успешной |
| *«Нет соединения с сервером»* | Проблемы с интернетом | Проверьте подключение к интернету и повторите попытку |
| *«Сессия истекла»* | Авторизация в системе истекла | Перезайдите в кабинет инструктора (заново введите email и пароль) |
| *«Ошибка API ключа Claude»* | Проблема конфигурации на стороне сервера | Обратитесь к администратору системы |

### Пустой или некачественный результат

| Проблема | Возможная причина | Решение |
|----------|-------------------|---------|
| Результатов 0 или очень мало | Текст слишком короткий или не по теме | Загрузите более объёмный и тематически релевантный текст |
| Вопросы слишком простые / сложные | Неправильно выбрана сложность | Измените уровень сложности и сгенерируйте заново |
| Препараты не найдены в тексте | В тексте нет упоминаний препаратов | Выберите только те типы контента, которые соответствуют содержимому лекции |
| Сценарии нереалистичные | Текст лекции не содержит клинических данных | Добавьте в пожеланиях конкретные требования к сценариям |

## 6. Ограничения текущей версии

### Что система НЕ умеет

- **Не распознаёт текст со сканов и изображений** (OCR не поддерживается). PDF должен быть текстовым.
- **Не обрабатывает таблицы и формулы** из документов — только обычный текст.
- **Не генерирует изображения или схемы.** Иллюстрации нужно добавлять вручную через раздел «Вопросы» или «Препараты».
- **Не сохраняет историю черновиков между сессиями на странице.** Если вы закроете страницу до утверждения — черновики останутся в базе, но для повторного доступа к ним потребуется помощь администратора.
- **Не позволяет редактировать уже утверждённые элементы** в разделе генерации. Для редактирования утверждённых элементов перейдите в соответствующий раздел (Вопросы или Препараты).
- **Не поддерживает генерацию на других языках** — только русский.
- **Ограничена длина входного текста** — максимум ~100 000 символов (около 30–35 страниц).
- **Не поддерживает загрузку нескольких файлов одновременно.** Загружайте по одному файлу за раз.

### Что планируется в будущих версиях

- **Структурированное тестирование (итерация 006):** инструктор сможет назначать сгенерированные тесты конкретным группам курсантов, открывать и закрывать модули, просматривать аналитику по результатам.
- **Полная миграция на собственную базу данных (итерация 007):** уход от зависимости внешних таблиц, единая система хранения.
- **Офлайн-режим (итерация 008):** работа без постоянного интернета с последующей синхронизацией.
- Возможности, которые находятся в бэклоге: дополнительные поля в интерфейсе вопросов и препаратов, улучшенная синхронизация.
